# 플라뇌르 — 배포 가이드

## 1단계: 로컬에서 먼저 테스트

파일을 브라우저에서 직접 열면(`file://`) JSON 로딩이 막힙니다.
`서버시작.bat`을 더블클릭해서 로컬 서버를 켜고 브라우저에서 `http://localhost:8000`을 열어주세요.

### 필요 조건 (둘 중 하나)
- **Python**: https://www.python.org/downloads/ → 설치 후 "Add to PATH" 체크
- **Node.js**: https://nodejs.org/ → LTS 버전 설치

---

## 2단계: GitHub 저장소 만들기

1. https://github.com 로그인
2. 오른쪽 상단 `+` → **New repository**
3. Repository name: `flaneur` (또는 원하는 이름)
4. Public 선택
5. **Create repository** 클릭

---

## 3단계: 파일 업로드

### 방법 A — 웹에서 직접 (쉬움)
1. 생성된 저장소 페이지에서 **uploading an existing file** 클릭
2. `D:\수창\플라뇌르\` 폴더 안의 파일들을 모두 드래그 앤 드롭
3. **Commit changes** 클릭

> 주의: 폴더째로 업로드하려면 폴더 안에 들어가서 파일만 선택하거나,
> 아래 방법 B를 사용하세요.

### 방법 B — GitHub Desktop (권장)
1. https://desktop.github.com 설치
2. **File → Add Local Repository** → `D:\수창\플라뇌르` 선택
3. Initialize Git Repository 클릭
4. **Publish repository** → GitHub에 업로드

---

## 4단계: GitHub Pages 활성화

1. 저장소 페이지 → **Settings** 탭
2. 왼쪽 메뉴 **Pages** 클릭
3. Source: **Deploy from a branch**
4. Branch: **main** / Folder: **/ (root)**
5. **Save** 클릭

약 2~3분 후 `https://[사용자명].github.io/flaneur/` 에서 사이트가 열립니다.

---

## 5단계: 관리자 패널 자동 발행 설정

GitHub Pages가 활성화되면 매주 JSON을 수동으로 업로드할 필요 없이
관리자 패널에서 직접 발행할 수 있습니다.

1. `https://[사용자명].github.io/flaneur/admin.html` 접속
2. **설정** 탭 클릭
3. GitHub 사용자명, 저장소 이름, Personal Access Token 입력
4. **설정 저장** → **연결 테스트**로 확인

### Personal Access Token 발급
1. GitHub → 우측 상단 프로필 → **Settings**
2. 왼쪽 하단 **Developer settings**
3. **Personal access tokens** → **Tokens (classic)**
4. **Generate new token (classic)**
5. Note: `플라뇌르 관리자`
6. Expiration: `No expiration` 또는 1년
7. Select scopes: **repo** 체크
8. **Generate token** → 토큰 복사 (다시 볼 수 없음!)

---

## 주간 업데이트 루틴

```
매주 금요일 또는 주말
  admin.html 접속
  → 소스 피드에서 항목 선택 및 요약 작성
  → 하단 "이번 주 발행" 버튼
  → 자동으로 GitHub에 반영 (GitHub Pages 빌드 약 1~2분)
```

---

## 도메인 연결 (선택)

커스텀 도메인(예: `flaneur.kr`)을 연결하려면:
1. 도메인 구매 (가비아, 후이즈 등)
2. GitHub Pages Settings → Custom domain에 도메인 입력
3. 도메인 업체 DNS에서 CNAME 레코드 추가:
   - Name: `www`
   - Value: `[사용자명].github.io`
4. `CNAME` 파일을 저장소 루트에 추가 (내용: 도메인 주소)

## 자주 묻는 문제

**Q: 사이트가 열리는데 소식이 안 보입니다.**  
A: `data/news.json`이 올바르게 업로드됐는지 확인하세요.

**Q: 관리자 패널에서 GitHub 발행이 실패합니다.**  
A: Token의 `repo` 권한을 확인하고, 사용자명/저장소명에 오타가 없는지 확인하세요.

**Q: 피드가 안 불러와집니다.**  
A: CORS 제한입니다. "직접 입력하기"로 내용을 입력하면 됩니다.
